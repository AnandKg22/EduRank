# Ideas Ledger

## 1. Purpose
Acts as the central repository for unvetted feature requests, "shower thoughts," and team suggestions. It keeps the core backlog clean while ensuring good ideas aren't forgotten.

## 2. Why It Matters
If developers dump every random idea into Jira/Linear, the backlog becomes a 1,000-ticket swamp. The Ideas Ledger acts as a holding pen where ideas can marinate before being formally accepted or rejected.

## 3. Example Structure
- **Idea**: Brief description.
- **Origin**: Who suggested it.
- **Status**: New, Under Review, Rejected, Promoted to Roadmap.

## 5. AI Usage Instructions
> [!NOTE]  
> If an AI agent generates an unsolicited idea during a brainstorming session, it should offer to log it in this `IDEAS.md` file rather than immediately writing code for it.

## 6. Developer Usage Instructions
- If you have an idea for a feature, add it here, not to the active sprint board.
- Review this list during "Hackathons" or "Cool-down" weeks between major sprints.

## 7. Best Practices
- **Do**: Briefly explain *why* an idea was rejected so it doesn't get suggested again 6 months later.
- **Don't**: Let the list grow infinitely. Purge stale ideas every year.

## 8. Maintenance Strategy
- **Owner**: Product Manager.
- **Update Frequency**: Continuous.
- **Trigger**: User interviews, team brainstorms, or random flashes of inspiration.
